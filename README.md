## 技能详情

### analysis-report

**触发词**：现状分析、AS-IS 分析、系统调研报告、技术评估、业务分析报告、风险评估报告、分析报告、帮我做个报告

生成专业的**暗色系单页 HTML 分析报告**，自带侧边栏导航和丰富的信息展示组件。

**核心特性：**

- 固定侧边栏导航，支持多章节锚点跳转
- 6 种语义色彩系统（青/绿/紫/琥珀/玫红/橙）
- 丰富组件库：信息卡片、数据表格、风险条目、时间轴、AS-IS vs TO-BE 对比表
- 支持现状分析、技术评估、业务可行性分析等多种报告类型
- 单 HTML 文件输出，无外部依赖（Google Fonts 除外）

### archify

**触发词**：架构图、系统图、流程图、时序图、数据流图、状态机、生命周期图、workflow、sequence、dataflow、lifecycle

功能最完整的架构图生成技能，支持 **5 种图表类型**，每个输出文件均内置：

- 🌓 深色 / 浅色主题切换
- 📤 PNG / JPEG / WebP / SVG 导出（4× 清晰度）
- 🎨 CSS 变量驱动的语义色彩系统

**支持的图表类型：**

| 类型             | 适用场景                                |
| ---------------- | --------------------------------------- |
| `architecture` | 系统组件、云架构、服务依赖、基础设施    |
| `workflow`     | 技术流程、审批流、CI/CD、运行手册       |
| `sequence`     | API 调用链、请求生命周期、服务交互      |
| `dataflow`     | 数据管道、ETL/ELT、数据血缘、PII 边界   |
| `lifecycle`    | 状态机、对象生命周期、运行/部署状态流转 |

渲染器位于 `archify/renderers/<type>/` 下，通过 JSON 描述文件驱动。

### architecture-diagram

**触发词**：架构图、系统架构、云架构、网络拓扑、技术组件关系图

基础版架构图技能，生成**暗色系独立 HTML 文件**（含内联 SVG）。适合快速生成简洁的系统架构说明图。

**特性：**

- 纯 SVG 绘图，无 JavaScript 依赖
- 语义色彩组件：前端/后端/数据库/云/安全/消息总线/外部
- 内置网格背景、组件框、安全组边界、区域边界

> 💡 如需主题切换和导出功能，请使用 `archify` 技能。

### customer_briefing_question_skill

**触发词**：帮我检查这份汇报、站在客户角度提前提问、模拟客户会怎么问、汇报材料审查、方案落地检查

在客户汇报、方案评审、售前交流、项目复盘前，**从客户视角模拟追问**，检查材料是否停留在概念层面。

**九大追问维度：**
业务真实性 · 数据来源 · 数据存储与数仓 · 加工治理 · 应用场景 · 技术实现 · 智能化能力 · 效果与价值 · 风险与不足

**三种汇报对象结构参考：**客户友好型 · 技术领导型 · 业务领导型

**输出内容：** 客户视角问题清单 + 汇报内容补强建议 + 可替换表述

### deep-business-analysis

**触发词**：帮我分析这个业务、这个系统有哪些痛点、帮我做业务调研、分析一下这个流程、做一个业务现状分析

面向业务架构师、产品经理、需求分析师的**系统性业务深度分析技能**（v1.1）。

**六阶段分析 SOP：**

1. 上下文输入与背景提取
2. 业务战略与目标定位
3. 业务现状结构化梳理（价值流 + 能力地图）
4. 多层次痛点诊断（现象→影响→根因 5 Why）
5. 痛点优先级矩阵（P0/P1/P2 加权评分）
6. 改进方向与实施路径建议

**输出格式：** Markdown 分析报告 + PlantUML 流程图代码

**适用行业：** 媒体内容生产、政务审批、制造业供应链、金融风控、医疗业务

### four-step-communication

**触发词**：帮我组织这段话、优化表达、这段汇报怎么改、融资路演、向老板汇报、跨部门沟通、客户谈判、四段式沟通

精通「四段式精准沟通法」的沟通策略顾问技能。

**四段框架：**

| 段落        | 别名          | 核心任务                 |
| ----------- | ------------- | ------------------------ |
| ① 结果开路 | 15 秒黄金窗口 | 明确量化目标，直击核心   |
| ② 现实铺底 | 事实支撑层    | 呈现关键数据与现状       |
| ③ 策略架桥 | 路径设计层    | 提出阶段性达成路径       |
| ④ 执行封顶 | 行动指令层    | 给出带时间节点的行动指令 |

**三种工作情形：** 优化已有内容 / 设计沟通脚本 / 解答理论问题

**主动诊断三大陷阱：** 关系幻觉、信息诅咒、模糊目标症

### meeting-minutes

**触发词**：会议纪要、纪要、整理会议、转写会议、会议记录整理、生成纪要、公文纪要

将口语化的会议记录自动转写为**正式公文风格的会议纪要**，符合央企/党政机关写作规范。

**规范要求：**

- 层级编号：`一、（一）1.` 三级结构
- 公文用语替换（口语→规范表述对照）
- 每个议题结构：背景/现状 → 讨论/决议 → 要求/后续安排
- 输出纯文本，无 markdown 符号，可直接粘入 Word/公文系统

**适用场景：** 内网项目例会、外网项目例会、专项工作会议、内部研讨会

### person-wiki

**触发词**：帮我分析这个人、建立人物档案、数字孪生、客户 Wiki、人物画像、这个人会怎么想、他下一步会怎么做

从录音转录、对话记录、会议笔记等原始素材中，自动提炼**结构化人物 Wiki**。

**七大维度：**

| 维度              | 内容                           |
| ----------------- | ------------------------------ |
| ◈ 基础身份       | 姓名、角色、组织、背景         |
| ⬡ 思维模式       | 思考风格、决策速度、信息偏好   |
| ◇ 价值观与优先级 | 核心价值观、目标、顾虑、驱动力 |
| ◎ 沟通风格       | 渠道偏好、语言风格、敏感话题   |
| △ 行为规律       | 决策触发条件、拖延模式、影响圈 |
| ○ 关系图谱       | 关系阶段、信任程度、关系杠杆点 |
| ◉ AI 预测参数    | 下一步行动预测、不确定因素     |

**输出内容：** 完整 Markdown Wiki + 可直接使用的 AI System Prompt（用于数字孪生模拟）

### pptx

**触发词**：deck、slides、presentation、.pptx、幻灯片、演示文稿、PPT

处理 `.pptx` 文件的完整技能，支持读取、编辑、从模板修改、从零创建。

**三种工作模式：**

| 任务          | 指南                                       |
| ------------- | ------------------------------------------ |
| 读取/分析内容 | `python -m markitdown presentation.pptx` |
| 编辑现有模板  | 参见 `editing.md`                        |
| 从零创建      | 参见 `pptxgenjs.md`（基于 pptxgenjs）    |

**设计系统：** 内置配色方案（10 套）、排版规范、布局选项、常见错误清单

**QA 流程：** 转换为图片 → 子 Agent 视觉检查 → 修复验证循环

**依赖工具：** markitdown · pptxgenjs · LibreOffice · Poppler

### startup-pressure-test

**触发词**：压力测试创业想法、验证问题、找前 10 个客户、定义 MVP、创业想法评估

用 Paul Graham 早期创业方法论**无情评估创业想法**，快速给出强/弱/转型判决。

**六种分析模式：**

| 模式                   | 说明                                |
| ---------------------- | ----------------------------------- |
| `pressure-test`      | 识别致命缺陷，给出裁决              |
| `problem-validation` | 验证痛点是否真实且紧迫              |
| `competition-map`    | 映射当前行为、直接/间接竞争者       |
| `first-10-customers` | 制定手动获取前 10 个客户的计划      |
| `mvp-plan`           | 定义 2 周内可验证核心假设的最小 MVP |
| `full`               | 以上所有模式的精简版合集            |

**输出内容：** 裁决（Strong/Weak/Pivot）+ 6 维评分卡 + 核心假设 + 致命缺陷表 + 前 10 客户行动计划 + MVP 定义

### 报价

**触发词**：报价单、报价表、工作量估算、人天、人月、功能报价、项目报价、系统改造报价

生成专业的**软件项目报价单 Excel 文件**（`.xlsx`），样式参考总台/广电行业软件项目报价惯例。

**表格结构：** 序号 / 一级功能（模块）/ 二级功能（子项）/ 功能描述 / 工作量（人天）

**输出规范：** 微软雅黑字体、合并标题、人天汇总行、专业配色
