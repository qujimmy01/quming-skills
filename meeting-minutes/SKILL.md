# 会议纪要转写 Skill

## 目标

将用户提供的会议过程记录（草稿、要点罗列、口语化记录等），转写为符合中央广播电视总台（或同等级别党政机关/央企）公文写作规范的正式会议纪要。

---

## 会议纪要结构规范

### 顶层结构

```
一、[项目名称/会议主题]
[召开时间]召开[会议名称]，[主持/参会/汇报方]汇报工作进展情况及计划安排，具体如下：
（一）[议题一标题]
[正文]
（二）[议题二标题]
[正文]
……

二、[第二个项目/主题]（如有）
……
```

### 层级编号规则

- 第一层：`一、二、三……`（大写汉字数字）
- 第二层：`（一）（二）（三）……`（括号汉字数字）
- 第三层：`1. 2. 3.`（阿拉伯数字加点）
- 禁止使用 markdown 符号（`#`、`*`、`-`）、英文标点替代中文标点

### 开头引语格式

- 标准句式：`[日期]召开[会议全称]，[参会主体]汇报工作进展情况及计划安排，具体如下：`
- 变体（如有多个参会主体）：`[参会主体A]、[参会主体B]参加会议，并听取[汇报方]汇报[事项]，对后续工作进行研讨。`

---

## 公文写作风格要求

### 用语规范

| 口语 / 草稿用词 | 公文规范替换 |
|---|---|
| 大家讨论了 | 会议听取了……并进行研讨 |
| 决定了 | 会议确定 / 会议一致同意 |
| 提出来了 | 项目组提出 / 会议提出 |
| 要做好…… | 需确保…… / 应……切实做好…… |
| 尽快 | 抓紧推进 / 尽快落实 |
| 安排下一步 | 对后续工作进行研讨部署 |
| 说明了情况 | 汇报了……工作情况 |
| 计划……去做 | 拟……推进 / 计划……完成 |
| 强调了重要性 | 会议强调，…… |
| 要求某方做某事 | 项目组要求，…… / 会议要求，…… |

### 句式特征

1. **被动与主动交替**：多用"会议听取了……""会议确定……""项目组要求……"等主语清晰的陈述句。
2. **四字格动词短语优先**：如"抓紧推进""稳步落实""平滑过渡""切实保障""综合评估"。
3. **数字用法**：具体数量用阿拉伯数字（42台、17台），序数和编号用汉字（三期终端、方案三）。
4. **时间表述**：日期格式为"X月X日（星期X）"；时间点用"X:00"；计划节点用"X月底前""本周内"。
5. **避免口语化**：去掉"其实""总的来说""基本上""大概"等模糊词。
6. **并列项目处理**：三项以上内容用"1.2.3."分条列出，两项以内可合并为一段。

### 结构完整性要求

- 每个`（X）`议题段落必须包含：**背景/现状 → 讨论内容/决议 → 工作要求/后续安排**，三者至少出现两项。
- 工作要求部分常见引导语：`项目组提出以下要求：` / `会议要求：` / `会议明确：`
- 后续安排部分常见引导语：`实施安排：` / `下一步：` / `拟……` / `计划……`

---

## 处理流程

1. **识别会议基本信息**：从输入中提取日期、会议名称、参会主体、汇报方。
2. **识别议题分类**：将内容划分为若干独立议题，每个议题对应一个`（X）`段落。
3. **逐议题结构化**：按"现状/背景 → 讨论/决议 → 要求/安排"三段组织内容。
4. **规范化语言**：将口语/简写替换为公文规范用语（参见上方对照表）。
5. **统一格式输出**：按层级编号规则输出纯文本，不使用任何 markdown 格式符号。

---

## 注意事项

- **保留原始数据**：数字、台数、时间节点、人名、系统名称等具体信息必须与输入一致，不得随意改动。
- **不补充臆造内容**：若输入缺少背景说明，可适当提炼，但不得凭空添加未出现的事实。
- **保持段落节奏**：各议题段落长度大体均衡，避免某一议题过于简短（少于3行）或过于冗长（超过20行），必要时可合并简短议题或拆分复杂议题。
- **输出为纯文本**：最终输出不含任何 markdown 符号，适合直接复制进 Word 或公文系统。
