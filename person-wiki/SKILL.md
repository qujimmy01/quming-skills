# Person Wiki Skill

## 目标

将非结构化的人物素材（录音、对话、笔记等）转化为结构化的人物Wiki，
最终输出可直接输入给AI的Markdown文档，用于数字孪生和行为预测。

---

## 工作流程

### Step 1：素材接收与评估

接收用户提供的素材，可以是以下任意形式：
- 录音转录文字
- 对话/聊天记录
- 会议纪要
- 邮件往来
- 个人观察笔记
- 多种素材混合

**评估素材质量：**
- 判断素材覆盖了哪些Wiki维度
- 识别明显的信息盲区
- 如果素材极少（< 200字），告知用户可补充哪些信息

### Step 2：深度提炼

仔细阅读所有素材，按照以下7个维度提炼信息。
**注意：不要照抄原文，要提炼出本质的规律和模式。**

读取提炼框架：参见本文件下方的「Wiki字段定义」部分。

**提炼原则：**
1. **推断优先于描述** — 不只记录"他说了什么"，而是推断"这说明他是什么样的人"
2. **模式优先于事件** — 找重复出现的行为/语言/思维模式，而非单次事件
3. **显性+隐性** — 既提炼明说的内容，也提炼言下之意、情绪信号、未说出口的顾虑
4. **置信度标注** — 对于推断性内容（非直接证据），用`[推断]`标注
5. **盲区标注** — 素材中缺失的重要维度，用`[待补充]`标注

### Step 3：生成Wiki文档

按照「输出格式」生成完整的Markdown Wiki。

### Step 4：生成AI模拟提示词

在Wiki末尾自动生成一段可直接使用的AI System Prompt，
让任何AI都能基于此Wiki模拟该人物的思维方式。

### Step 5：给出补充建议

列出2-3个最关键的信息盲区，建议用户补充哪些素材可以让Wiki更准确。

## Wiki字段定义

### 维度一：基础身份

| 字段 | 提炼要点 |
|------|---------|
| 姓名/称呼 | 全名 + 对方如何称呼自己/被称呼 |
| 角色定位 | 在此关系中的角色（客户/同事/合作伙伴/上级等） |
| 所属组织 | 公司、部门、职位 |
| 背景简述 | 教育、行业经历、可推断的人生阶段 |

### 维度二：思维模式

| 字段 | 提炼要点 |
|------|---------|
| 思考风格 | 数据驱动？直觉型？系统化？跳跃性？从细节到全局还是反之？ |
| 决策速度 | 快速拍板 / 需要消化时间 / 反复权衡 / 依赖他人确认 |
| 信息偏好 | 偏好数字、案例、类比、故事、图表中的哪种？ |
| 常用思维框架 | 习惯用什么框架看问题？（ROI、风险优先、关系优先、先例参照等） |

### 维度三：价值观与优先级

| 字段 | 提炼要点 |
|------|---------|
| 核心价值观 | 效率/稳定/创新/关系/面子/成本/品质... |
| 当前首要目标 | 现在最在乎什么？想解决什么核心问题？ |
| 核心顾虑/规避点 | 什么会让他说No？最怕哪类风险？ |
| 驱动力/兴奋点 | 什么让他眼睛发亮？什么会触发他的Yes？ |

### 维度四：沟通风格

| 字段 | 提炼要点 |
|------|---------|
| 偏好沟通渠道 | 微信/电话/邮件/当面/视频... |
| 语言风格 | 正式/随意、简短/详尽、直接/迂回、情感型/理性型 |
| 回应模式 | 通常怎么回应？快/慢？主动/被动？喜欢确认还是推进？ |
| 敏感话题/禁区 | 哪些话题要避开？什么让他不舒服或防御？ |
| 标志性用语 | 常用词、口头禅、惯用表达（直接引用原文） |

### 维度五：行为规律

| 字段 | 提炼要点 |
|------|---------|
| 决策触发条件 | 什么情况下他会推进/同意？需要什么条件满足？ |
| 拖延/回避模式 | 不想推进时会怎么做？有哪些信号？ |
| 影响圈 | 谁影响他的决定？他在乎谁的看法？ |
| 历史决策记录 | 重要决定的模式总结 |

### 维度六：关系图谱

| 字段 | 提炼要点 |
|------|---------|
| 当前关系阶段 | 初识 / 建立信任中 / 深度合作 / 有摩擦 / 长期稳定 |
| 信任程度 | 低（谨慎）/ 中（有基础）/ 高（相互信任）/ 极高（战略伙伴） |
| 关键共同经历 | 重要的合作、冲突、里程碑 |
| 关系杠杆点 | 什么加深了关系？什么可能损害关系？ |

### 维度七：AI预测参数

| 字段 | 提炼要点 |
|------|---------|
| 下一步最可能的行动 | 基于当前状态，他最可能做什么？ |
| 行为预测逻辑 | 支撑上述预测的具体依据 |
| 关键不确定因素 | 哪些变量会改变预测结果？ |
| 信息盲区 | 还不了解他的哪些方面？ |

## 特殊场景处理

### 素材极少时
如果素材不足以支撑某个维度，不要编造，而是：
- 标注 `[待补充：建议通过X方式获取此信息]`
- 在「补充建议」中优先列出

### 素材矛盾时
如果不同素材呈现矛盾信息：
- 两种情况都记录，标注`[存在矛盾：素材A显示...，素材B显示...]`
- 给出可能的解释（情境不同？时间不同？）

### 多次会话积累时
如果用户提供多批次素材更新Wiki：
- 先读取已有Wiki
- 标注新增/变更的字段
- 更新「素材来源」和「生成时间」

### 敏感信息处理
如果素材涉及极度隐私内容（健康、财务等）：
- 只提炼与行为预测相关的部分
- 不逐字记录原始敏感细节
